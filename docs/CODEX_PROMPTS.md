# Идеи промтов для развития Personal Portal

## 1. Синхронизация задач и постов с бэкендом
```
Выступай как senior full-stack разработчик. В репозитории personal-portal нужно реализовать хранение задач (src/pages/Home.jsx) и заметок (src/pages/Posts.jsx) на бэкенде. Создай REST API на Express с таблицами в PostgreSQL, перенеси текущую работу с localStorage на fetch-запросы и обеспечь optimistic UI. Подробно опиши миграции, DTO и точки интеграции с существующим AuthContext (src/context/AuthContext.jsx).
```

## 2. Интеграция Outline API для VPN ключей
```
Представь, что ты backend engineer. Добавь в раздел VPN (src/pages/vpn/Outline.jsx) реальную работу с Outline API: вывод списка ключей, создание и удаление. Реализуй сервис в backend/routes/vpn.js, используй токен Outline из .env и обеспечь кеширование ответов. На фронтенде создай хук useOutlineKeys и обнови UI, чтобы поддерживались состояние загрузки и ошибки.
```

## 3. Расширенная аналитика контента
```
Ты работаешь как product analytics разработчик. В разделе /analytics добавь фильтрацию и поиск по данным, получаемым из таблицы content_items. Реализуй API endpoint /api/content/analytics, который возвращает статьи с тегами и статистикой просмотров. На фронтенде внедри контекст с кэшированием, skeleton-состояния и сортировку по популярности.
```

## 4. Уведомления и дайджесты
```
Действуй как frontend engineer. Реализуй центр уведомлений в Personal Portal: колокольчик в Header, выезжающую панель и хранение непрочитанных событий в IndexedDB. Подключи push-уведомления через Service Worker, добавь API /api/notifications и настрой отображение журнала событий из backend/routes/admin.js.
```

## 5. Ролевая модель с гранулярными правами
```
Работай как архитектор безопасности. Усиль контроль доступа: добавь таблицу permissions и RBAC-слои в backend/middleware/auth.js. Реализуй UI в admin/users для управления отдельными правами (чекбоксы) и адаптируй маршрутизацию в src/App.jsx, чтобы разделы проверяли наличие конкретного permission вместо жёстко заданной роли.
```
